# AI 调用日志说明

现在系统会输出详细的调用日志，帮助你理解整个 AI 评分和讨论过程。

## 日志格式

### 阶段一（评分阶段）

每个评委的调用会输出：
```
================================================================================
[阶段一] 创建评委: grok_judge (Grok)
模型: gpt-4o-mini
System Message 长度: 2345 字符
================================================================================
```

每个评委的响应会输出：
```
================================================================================
[阶段一] 评委 grok_judge 原始输出:
<inner_monologue>
这是什么玩意儿...
</inner_monologue>
```json
{
  "judge_id": "grok_judge",
  ...
}
```
================================================================================
```

### 阶段二（讨论阶段）

讨论开始时会输出：
```
================================================================================
[阶段二] 群聊开始
初始上下文:
【本轮参赛作品信息】
类型：outfit
ID：test_001

【各评委首轮亮分与核心观点】
▼ Grok (打分: 2.5)
  观点: "这简直是灾难..."
...
================================================================================
```

每一轮发言会输出：
```
================================================================================
[第 1 轮] 发言者: grok_judge
完整回复:
@ChatGPT-5，你丫虚伪到家了吧？...
================================================================================
```

## 如何使用

1. 运行你的评分流程
2. 查看终端输出，所有日志会按时间顺序显示
3. 搜索 `[阶段一]` 查看评分过程
4. 搜索 `[阶段二]` 或 `[第 X 轮]` 查看讨论过程
5. 每个分隔线（`===`）标记一个完整的 AI 调用

## 注意事项

- System Message 默认只显示长度，如需查看完整内容，取消 `stage_one.py` 中的注释
- 原始输出包含 `<inner_monologue>` 和 JSON 部分
- Selector 的选择决策是自动的，会在每轮发言的 `发言者` 字段中体现
